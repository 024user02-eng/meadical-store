Medical Store Management System SRS Outline 
1. Introduction
1.1 Purpose: To define the scope, features, and constraints of the Medicine Shop Software (MSS).
1.2 Scope: A system to automate inventory, billing, and accounting, reducing manual effort.
1.3 Definitions & Acronyms: E.g., POS (Point of Sale), Generic Name, Expiry Date.

2. Overall Description
2.1 Product Perspective: Independent application or web-based system connecting pharmacy staff, admin, and potentially customers.
2.2 Product Functions:
Inventory Control: Add/Update/Delete medicine stock, tracking low stock, and managing batch numbers/expiry dates.
Billing/Sales: Generating invoices, managing payments, and customer records.
Supplier Management: Managing creditor data, purchase orders, and supplier info.
Reporting: Sales reports, inventory reports, and profit/loss analysis.
2.3 User Classes and Characteristics:
Administrator: Full access to reports, inventory, and staff management.
Staff/Pharmacist: Access for billing, searching stock, and updating sales.
2.4 Operating Environment: Windows/Linux server, modern web browsers (Chrome, Firefox), or local desktop environment.
2.5 Design and Implementation Constraints: Data security (patient confidentiality), regulatory compliance, and hardware limitations.

3. Functional Requirements
3.1 User Authentication: Secure login for Admin and Staff.
3.2 Search Functionality: Search by name, category, or manufacturer.
3.3 Inventory Alert: Notifications for expired medicines or low stock levels.
3.4 Report Generation: Monthly sales, profit reports, and inventory status.
